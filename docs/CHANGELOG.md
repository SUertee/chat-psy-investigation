# 项目变更日志

## 2026-03-08（本轮会话）

### 1) 实验流程与 Probe（实验组/对照组）
- 新增 `probe1`~`probe6`（两题、1-9 滑条），并接入实验组与对照组。
- 时间点已对齐为：
  - `probe1`：前测后；
  - `probe2`：AI Tutor 后（练习一提示前）；
  - `probe3/5`：每轮练习结束后、反馈前；
  - `probe4/6`：反馈阅读后。
- 保留 `SECOND_CLIENT`（第三次练习）与 `questionnaire3` 流程，不追加 probe。

### 2) 对照组配对与 fallback 机制
- 新增配对等待超时参数：`MATCH_TIMEOUT_MS = 300000`（5 分钟）。
- 实现对照组配对超时 fallback：
  - 标记 `control_timeout_fallback` 及原因/时间/发生轮次/等待时长；
  - 对照组流程内可回退到 AI 对话模式（不改变原始分组记录）。
- 修复过一次分支串线后，调整为：
  - 实验组分支仅 `group === 'experimental'`；
  - 对照组 fallback 仅在对照组分支内处理，避免流程互串。

### 3) 对照组“共同阅读时间”强同步与启动条件
- 新增后端同步字段：`read_deadline_at`、`server_now`、`shared_read_seconds`。
- 两端计时统一改为“按同一后端截止时间计算”，不再本地各自起算。
- 修正倒计时启动条件为“双方都提交后才开始”：
  - 来访者提交反馈后，先等待咨询师提交评估报告；
  - 咨询师提交评估报告后，若来访者已提交，才生成共同阅读倒计时；
  - 咨询师先提交同理，等待来访者提交后再启动计时。
- 新增后端接口：`/rooms/{room_id}/counselor-report-submitted`。

### 4) 对照组超时结束行为
- 修复咨询师端 10 分钟到时行为：
  - 到时自动结束当前轮（不再弹确认框阻塞）；
  - 直接进入后续评估/反馈流程。

### 5) 数据导出增强
- `event_log` 与 `participant_wide` 增加 `probe1`~`probe6`固定列（含提交时间）。
- 导出中新增 fallback 相关字段：
  - `initial_group`
  - `control_timeout_fallback`
  - `control_timeout_fallback_reason`
  - `control_timeout_fallback_at`
  - `control_timeout_fallback_prompt_key`
  - `control_timeout_fallback_wait_ms`

### 6) UI 文案与展示调整
- 浏览器推荐文案防换行处理。
- 流程卡片去除各步骤括号时间显示。
- 实验总时长文本统一为 `90分钟`（说明页 + 知情同意）。
- 视频窗口放大并改为按源比例显示（`contain` + 黑底），减少放大后异常显示。
- 对照组来访者反馈等待页标题/结构与咨询师侧对齐（第一部分一致，第二部分保留角色差异）。

### 7) 问卷内容更新（最后一页）
- `questionnaire3` 替换为“实验后感受与支持需求”：
  - 情绪不适（是/否）；
  - 若“是”显示是否希望研究团队联系（是/否）；
  - 若再“是”显示联系方式输入（可选）；
  - 热线说明固定显示（与前序选择无关）。
- 问卷渲染器新增：
  - `visible_if` 条件显示能力；
  - `info` 说明块题型；
  - `text` 文本输入题型。
- 确认联系方式为可选，正式实验不填写可直接提交。

### 8) 后测文案统一
- 后测中“技能练习”相关表述统一替换为“模拟练习”。
