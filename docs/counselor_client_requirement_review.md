# 咨询师/来访者双端需求评审与实施方案

更新日期：2026-03-07 22:37

## 1. 需求目标（原始要求整理）

### A. 咨询师界面（Counselor View）
- UI 与实验组对话界面保持一致。
- 左侧显示来访者（Peer B）消息，白色气泡。
- 右侧显示咨询师自己消息，绿色气泡。
- 右上角保留粉色按钮：`我已完成评估，结束此阶段`。

### B. 来访者界面（Client View，翻转版）
- 视觉翻转以模拟来访者视角。
- 右侧显示来访者自己消息，绿色气泡。
- 左侧显示咨询师（Peer A）消息，白色气泡。
- 输入区右侧或上方常驻角色脚本（Profile），用于随时查看身份背景、压力来源、危机细节。
- 来访者界面不提供粉色结束按钮，结束权限仅归咨询师。

### C. 核心交互流程
1. 对话开始自动发送固定开场白（来自来访者），例如：`你好… 我觉得好难受，能跟你聊聊吗……`
2. 双方实时打字沟通。
3. 咨询师点击粉色按钮结束评估后：
- 咨询师跳转到“咨询记录”填写页面；
- 来访者跳转到“反馈”填写页面。

### D. 来访者反馈页结构
- 左侧：与咨询师聊天记录（只读）。
- 右侧：反馈表格（可填写）。


## 2. 当前实现状态（Review 结论）

## 2.1 已实现部分
- 聊天基础布局、左右气泡和颜色风格已具备（左白右绿）。
- 前后端配对聊天可发送与轮询拉取消息，具备实时通信基础能力。

## 2.2 未实现/不满足部分
- 未实现“仅咨询师可结束”：粉色结束按钮是全局按钮，当前聊天流程对双方都可见。
- 未实现“自动来访者开场白”：进入配对后显示的是系统提示文案（连接中、角色说明），不是固定来访者开场句。
- 未实现“结束后分流”：当前流程没有 counselor/client 两条后续路径（咨询记录页 vs 反馈页）。
- 未实现“来访者常驻 Profile”：聊天界面未挂载脚本说明面板。
- 后端轮次推进机制是“双方都 ready 才推进”，与“咨询师单方结束权”不一致。
- 当前配置默认强制实验组（`GROUP_MODE = 'experimental'`），对照组双人流程不会自然触发。


## 3. 实施方案（完整可执行）

### 阶段 1：角色与权限落地（必须先做）
- 前端在进入房间后计算当前轮次角色：`isCounselor` / `isClient`。
- 粉色结束按钮显示规则改为：仅 `isCounselor = true` 时显示。
- `isClient` 时强制隐藏结束按钮，并且不能调用结束接口。

### 阶段 2：自动开场白机制
- 在后端房间轮次中新增“开场白是否已注入”状态。
- 进入轮次时若未注入，则系统生成一条固定开场白消息（`sender_role = client`）。
- 两端进入聊天后都能看到这条第一句。

### 阶段 3：来访者视角与脚本挂载
- 聊天 UI 增加 `viewMode`（`counselor` / `client`）。
- 在 `client` 视图内增加常驻 Profile 区域（输入框上方或右侧），显示该轮角色档案与风险细节。
- 保持消息规则：自己永远右侧绿色，对方永远左侧白色。

### 阶段 4：结束流程重构（核心）
- 咨询师点击结束后，调用后端“结束本轮”接口，房间标记 `ended_by_counselor`。
- 来访者轮询到房间已结束后，自动收口聊天并跳转反馈页面。
- 不再依赖“双方都 ready 才结束”。

### 阶段 5：结束后分流页面
- 咨询师端：进入“咨询记录表单”页面并提交。
- 来访者端：进入“左聊天记录 + 右反馈表格”页面并提交。
- 两种提交结果都纳入导出数据结构。

### 阶段 6：时间线与导出对齐
- 按角色拆分聊天后续 timeline。
- 导出数据中新增：
- `counselor_record`
- `client_feedback`
- `round_end_reason`
- `ended_by`


## 4. 验收标准（Done Criteria）

- 同一房间内，咨询师能看到粉色结束按钮，来访者看不到。
- 对话开始时，首条消息是固定来访者开场白。
- 双方都可实时收发消息。
- 咨询师点击结束后，双方都退出聊天且分流到各自页面。
- 来访者反馈页布局为“左聊天记录、右反馈表格”。
- 角色脚本在来访者界面常驻可见。


## 5. 建议实施顺序

1. 先改权限与结束逻辑（阶段1+4）。
2. 再补自动开场白（阶段2）。
3. 再做来访者 Profile 和反馈页布局（阶段3+5）。
4. 最后统一 timeline 与数据导出（阶段6）。

### 6.1 实验流程与界面
- 重构练习前引导页：`图1` 改为角色中性/对象中性文案；`图2` 改为连接成功后展示角色与任务说明。
- 对照组配对流程从聊天页中拆出，先完成配对与角色确认，再进入统一聊天界面。
- 统一练习标题与流程：练习三使用与前面一致的图1/图2样式与标题（`练习3`）。
- 统一粉色按钮文案为：`我已完成风险评估，结束此阶段`（实验组/对照组/三次练习一致）。
- 扩宽聊天区并统一实验组/对照组、咨询师/来访者宽度表现，改善不同屏幕下可读性。

### 6.2 对照组双人同步与计时
- 新增“配对成功即下发统一开始时间”机制：服务器给出 `start_at = now + 30s`，双端倒计时后同时入聊。
- 修复双端倒计时不同步问题：将关键阶段改为“等待态 + 共同阅读态”双阶段计时，避免单端先跑表。
- 修复超时自动提交 bug：来访者反馈/咨询师评估超过 5 分钟自动提交，不再出现“重置重新计时”。

### 6.3 反馈与复盘流程
- 对照组改为“咨询师点结束即结束本轮”，来访者立即进入反馈填写，不再依赖咨询师先交记录。
- 对照组反馈结构改为与实验组一致：维度1-3仅保留“做得好的地方/需要提升的地方”，维度4为单一综合建议。
- 增加共同阅读阶段：双方都能看到反馈内容，咨询师完成阅读后推进下一阶段。
- 来访者等待页升级为两部分结构（共同阅读提示 + 已提交反馈详情），并显示“共同阅读剩余”。

### 6.4 个案与提示词一致性
- 对照组来访者脚本改为与 `ai.js` 对应版本一致：第一轮使用低风险画像，第二轮使用高风险画像；右侧标题统一为`来访者脚本`，不显示“小B/低风险”等标签。
- 修复练习三个案映射错误：确保练习三使用中风险档案（不显示姓名标签）。
- 去除 AI 反馈中的“评估准确性”板块，并统一反馈文字为黑色风格；随后按需求恢复复盘页彩色视觉。

### 6.5 时间线与重复环节修复
- 删除练习三之后重复出现的“风险评估表 + AI反馈”冗余节点，避免实验组/对照组重复弹出。
- 增强多处弹窗与提交流程的防重入保护（避免重复弹窗、重复提交、重复推进）。

### 6.6 数据存储与导出（重点）
- 扩展导出为双表结构：
  - `event_log`：长表（事件级，便于审计）
  - `participant_wide`：宽表（每被试一行，便于 Python 合并分析）
- 新增并补齐关键数据保存项：
  - AI对话、真人配对对话、AI答疑、风险评估、同辈反馈、AI督导反馈、问卷原始作答
  - 分组与流程信息（实验/对照、对照组先后角色）
  - 关键时长（前后测、各次对话、风险评估填写、反馈填写）
- 修复历史遗漏：对照组 `pairedChatHistory` 不再在每轮初始化时被清空，确保完整保留全程消息。

### 6.7 分半测验随机化（最新）
- 前后测分半题组新增独立随机化（与实验/对照分组随机独立）：
  - 被试可能 `A→B`，也可能 `B→A`
- 保存随机顺序到数据导出字段，便于后续统计控制：
  - `split_half_order`
  - `pretest_half`
  - `posttest_half`
